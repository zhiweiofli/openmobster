In the spirit of confidence building when using a new technology, lets start with a simple app. We will call it the HelloMobster moblet.

2.1.1. System Requirements

2.1.1.1. Cloud Server

  • Java SE JDK v6.0 (the server component is version 5.0 compliant, but the Blackberry Development Environment requires 6.0)

2.1.1.2. Blackberry Development

  • Java SE JDK v6.0
  • Eclipse 3.4 (the JDE plugin does not run on 3.5 yet)
  • Windows® 2000 SPI or later, or Windows XP
  • Windows Vista (BlackBerry JDE v4.2.1 and higher)
  • Monitor with a resolution of 1024 x 768 or higher

    2.2. Create your first Mobile Cloud App (a.k.a Moblet)

    Download an OpenMobster distribution from here: Downloads .

    In your distribution, go to the directory AppCreator. Inside the directory, use a tool called the 'appcreator.bat' (Windows), 'appcreator.sh' (Linux and Mac) to generate a skeleton 'Moblet' project 

    				
		appcreator.bat

    This will generate a Maven-based skeleton for the Mobile App. Each generated project has the following three maven modules:

    • app-rimos: Contains the device side application code for the Blackberry OS - version 4.3.0 and higher
    • cloud: Contains the "OpenMobster Cloud Server" based artifacts which will be deployed on the server side
    • moblet: Represents a "OpenMobster Moblet" which combines both the device side and server side artifacts into one single artifact. The moblet is deployed as a simple jar file into the "OpenMobster Cloud Server". When the moblet is deployed into the Cloud server it is registered with the built-in App store. Once registered with the App Store, this moblet can be easily downloaded, installed, and managed on the device

    2.3.  Run the Cloud Server in Development Mode

    Go to the 'cloud' module 

    				
		cd cloud

    Run the Cloud Server in Development Mode 

    				
		mvn -PrunCloud integration-test

    This will start the Cloud Server and the server artifacts corresponding to this project. This is a fully functional instance of the Cloud Server ready to process traffic from your moblets.

    2.4.  Build and Deploy the Moblet in a Blackberry Simulator

    Go to the 'app-rimos' module 

    				
		cd app-rimos

    Build and deploy the Moblet into the Simulator 

    				
		mvn install

    To improve development productivity "mvn package or mvn install" command installs the required MobileCloud runtime into the specified Blackeberry Simulator. This binary improves developer productivity by automating the manual provisioning processes by automagically creating a Cloud account under the name: blah2@gmail.com .

    Note: This is a strict development stage only optimization and should not be used in a real world setting. The location of the simulator is specified in the RIM_JDE_HOME environment variable.

    2.5. Run the Moblet in the Simulator

    Run the Blackberry Simulator. It will take a while and once it loads, the Moblet and the MobileCloud should be installed and ready for use. You can go to the Apps and there should be an icon for HelloMobster . Click the icon. You should see the following screens:

    2.6. Making some changes

    Inside the 'cloud' module, go to org.openmobster.core.examples.rpc.DemoMobileBeanService.java file and modify the invoke implementation as follows:

    Change the line 

    				
response.setAttribute(name, "response://"+value);

    to 

    				
response.setAttribute(name, "response://hellomobster/"+value);

    Re-start the 'Cloud Server'

    2.7. Run in Production Mode

    More details about the OpenMobster Production Mode setup can be found here: Production Mode

    Inside the 'moblet' module, execute: 

    				
		mvn install

    This will generate a target/${project.name}-${project.version}-moblet.jar . Copy this file to your JBoss 'openmobster' instance at openmobster/deploy directory. Run the moblet in the simulator
